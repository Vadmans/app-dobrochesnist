# Доброчесність — мобільний застосунок (Android APK)

Це готовий проєкт: ваш прототип, загорнутий у **Capacitor**. Після кількох команд
ви отримаєте файл `.apk`, який встановлюється на Android без публікації в Play Market.

> APK збирається **на вашому комп'ютері** — у середовищі, де його готували, немає
> Android-інструментів та інтернету. Нижче все, що для цього треба.

---

## 1. Що встановити заздалегідь (один раз)

- **Node.js 18+** — https://nodejs.org
- **Android Studio** — https://developer.android.com/studio
  (під час встановлення лишіть галочку **Android SDK**)
- **JDK 17** — зазвичай ставиться разом з Android Studio

Після встановлення Android Studio хоча б раз відкрийте його, щоб він
дозавантажив SDK-компоненти.

---

## 2. Збірка (у теці проєкту)

```bash
# 1. встановити залежності
npm install

# 2. зібрати веб-частину (створює теку dist/)
npm run build

# 3. створити нативний Android-проєкт (лише першого разу)
npx cap add android

# 4. перенести зібраний застосунок у нативний проєкт
npx cap sync android
```

Далі — два способи зробити сам APK.

### Спосіб А — через Android Studio (простіше)

```bash
npx cap open android
```

У вікні, що відкриється: меню **Build → Build Bundle(s) / APK(s) → Build APK(s)**.
Коли збірка завершиться, з'явиться повідомлення з посиланням **locate** — там і буде `.apk`.

### Спосіб Б — командою (без відкриття студії)

```bash
cd android
./gradlew assembleDebug        # Windows: gradlew.bat assembleDebug
```

Готовий файл:
`android/app/build/outputs/apk/debug/app-debug.apk`

---

## 3. Встановлення на телефон

Перекиньте `app-debug.apk` на Android-телефон (через кабель, месенджер чи папку),
відкрийте його та підтвердьте. За потреби увімкніть у налаштуваннях телефона
**«Встановлення з невідомих джерел»** для застосунку, яким відкриваєте файл.

---

## 4. Що далі (за потреби)

- **Іконка та назва.** Іконку міняють у `android/app/src/main/res` (Android Studio:
  правою кнопкою на `res` → New → Image Asset). Назву — у `capacitor.config.json` (`appName`).
- **Підписаний реліз-APK** (для роздачі в організації) — `./gradlew assembleRelease`
  з ключем підпису. Знадобиться, коли вийдете за межі тестування.
- **Push-нагадування** на телефоні — додається плагіном `@capacitor/local-notifications`
  (локальні нагадування за датою події) або `@capacitor/push-notifications` (серверні).

> Зараз дані застосунку демонстраційні й тримаються в пам'яті. Коли під'єднаєте
> бекенд (PostgreSQL + API), застосунок підтягуватиме реальні події звідти —
> сам мобільний код для цього майже не зміниться.
