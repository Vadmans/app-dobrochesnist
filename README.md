# Доброчесність

## Загальний опис

**Доброчесність** — це мобільний додаток та веб-платформа, призначені для інформування працівників про строки та вимоги антикорупційного законодавства України.

Основна мета системи — забезпечення своєчасного інформування користувачів щодо необхідності виконання антикорупційних заходів, подання декларацій, повідомлень про суттєві зміни майнового стану, дотримання обмежень та інших вимог законодавства.

Система дозволяє централізовано адмініструвати події, надсилати push-повідомлення, надавати довідкову інформацію та забезпечувати комунікацію між користувачами й адміністраторами.

---

# Основні можливості

## Календар подій

Система містить інтерактивний календар подій, який дозволяє:

* переглядати майбутні строки;
* відображати події за місяцями;
* отримувати інформацію про найближчі строки;
* експортувати події до календаря пристрою;
* переглядати події за категоріями;
* отримувати автоматичні нагадування.

### Категорії подій

* Декларування;
* Конфлікт інтересів;
* Подарунки;
* Повідомлення;
* Навчання;
* Обмеження.

---

## Push-повідомлення

Реалізована інтеграція з **Firebase Cloud Messaging (FCM)**.

### Функціонал

* реєстрація мобільних пристроїв;
* автоматичне збереження Firebase Token;
* централізована відправка повідомлень;
* інформування про нові події;
* нагадування про строки виконання вимог законодавства;
* повідомлення про відповіді адміністратора в чаті.

### Переваги

* миттєве інформування користувачів;
* мінімізація ризику пропуску важливих строків;
* централізоване керування повідомленнями.

---

## Локальні нагадування

Для кожної події можуть бути встановлені нагадування:

* за 30 днів;
* за 14 днів;
* за 7 днів;
* за 3 дні;
* у день події.

Нагадування працюють навіть без відкритого додатка.

---

## Довідник

У додатку реалізовано окремий розділ довідкової інформації.

### Можливості

* розміщення методичних рекомендацій;
* посилання на нормативно-правові акти;
* розміщення внутрішніх документів;
* швидкий доступ до корисних ресурсів.

---

## Чат із адміністрацією

Система містить вбудований модуль запитань та відповідей.

### Користувач може

* поставити питання адміністратору;
* переглядати історію своїх звернень;
* отримувати повідомлення про нові відповіді.

### Адміністратор може

* переглядати всі звернення;
* надавати відповіді;
* редагувати відповіді;
* видаляти звернення.

---

## Інтерактивний помічник

Передбачений модуль підтримки прийняття рішень.

Дозволяє користувачу:

* пройти покрокове опитування;
* визначити необхідність подання повідомлення;
* отримати рекомендації щодо подальших дій;
* швидко знайти необхідну інформацію.

---

# Адміністративна панель

Адміністративна панель забезпечує централізоване керування системою.

## Управління подіями

* створення подій;
* редагування подій;
* видалення подій;
* визначення категорій;
* встановлення строків;
* налаштування нагадувань.

## Управління довідником

* додавання матеріалів;
* редагування матеріалів;
* видалення матеріалів;
* додавання зовнішніх посилань.

## Управління адміністраторами

* створення облікових записів;
* зміна паролів;
* блокування доступу;
* контроль авторизації.

## Управління пристроями

* перегляд зареєстрованих пристроїв;
* перегляд Firebase Token;
* контроль активності пристроїв;
* видалення застарілих пристроїв.

## Push-розсилка

Адміністратор може:

* створювати повідомлення;
* визначати заголовок повідомлення;
* визначати текст повідомлення;
* надсилати повідомлення всім користувачам.

# Архітектура системи

## Frontend

Мобільний додаток:

* React
* Capacitor
* JavaScript
* Lucide Icons

## Backend

Серверна частина:

* Python
* FastAPI
* SQLAlchemy
* Uvicorn

## База даних

Використовується PostgreSQL.

Основні таблиці:

* admin_users
* events
* reference
* devices
* chat_messages

---

# Хмарна інфраструктура

## Render

Використовується для:

* розміщення бекенду;
* запуску API;
* адміністрування системи;
* обробки push-повідомлень.

## Neon

Використовується для:

* зберігання даних;
* резервування інформації;
* забезпечення роботи PostgreSQL.

## Firebase

Використовується для:

* push-повідомлень;
* реєстрації пристроїв;
* доставки повідомлень.

# Безпека

У системі реалізовано:

* авторизацію адміністраторів;
* захищені сесії;
* хешування паролів (PBKDF2 SHA-256);
* захист від перебору паролів;
* захист API;
* контроль доступу;
* централізоване керування користувачами;
* захищене зберігання даних;
* використання HTTPS.

# Автор проєкту

Проєкт розроблено для автоматизації процесів інформування та контролю виконання вимог антикорупційного законодавства.

Основна мета системи — підвищення рівня обізнаності користувачів, своєчасне інформування та мінімізація ризику порушення встановлених строків.
